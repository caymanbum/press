# Backport the Crash Reporting System from Iguana X to Iguana 6

I know that, out of excessive paranoia, one employee thought it was a good idea to make Iguana 6 generate strange binary files. This led to a highly manual process to reproduce crash information using an outdated version of Microsoft Visual Studio.

This is actually a form of corporate insanity.

Well, that’s what you’d expect from paranoia!

We need to take the sane crash reporting process from Iguana X and implement it in Iguana 6, preferably outputting reports in a simple format such as GITHUB issues, rather than creating a manual filing process.

This way affiliates have the opportunity to compete and demonstrate real value by finding real long term solutions to these problems so we can become the most reliable platform for middleware on the planet.

Simple insane goal.  [Extremes matter](/system/extreme).
