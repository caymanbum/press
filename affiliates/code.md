# Consulting Code Quality

In the past, we have occasionally encountered situations where the quality of code delivered by consulting partners did not fully meet our expectations, which risks affecting the overall perception of our product. We fully recognize that developing high-quality code is a challenging task—especially when consultants are often working within tight project timelines and other constraints.

With this in mind, I’d like to propose a practical and collaborative solution.

Going forward, all [affiliate](i.md) consultants will be asked to share their source code with Eliot, along with suitable sample data to ensure the code can be run and tested effectively.

This initiative is intended to support the development of AI-assisted workflows that will help both Eliot and our consulting partners to streamline, improve, test, and secure project code. The purpose here is not to introduce an onerous review process or assign blame; rather, our aim is to foster an open feedback loop that elevates the quality of everyone’s work.

In our experience, the best code is straightforward, clearly solves the problem at hand, and is easy to understand, maintain, and secure. Simplicity not only enhances clarity but also makes systems safer and easier to operate.

Our ultimate objective is mutual improvement: consultants benefit from better tools and constructive feedback, customers enjoy higher-quality implementations, and our product is consistently represented by code we can all stand behind.

Through this collaborative approach, we are confident that both security and overall code quality will be enhanced for everyone involved.

There are some [tools that Eliot will develop for this purpose](tools.md).
